# اهورا – پخش موبایل

> وب‌اپلیکیشن (PWA) مدیریت پخش/فروش سیار: لیست قیمت، سفارش‌گیری، مشتریان، بازدید ویزیتورها و گزارش‌گیری — همه در یک فایل HTML، با همگام‌سازی ابری real-time و پشتیبانی کامل از حالت آفلاین.

## معرفی

اهورا یک PWA تک‌صفحه‌ای برای مدیریت یک کسب‌وکار پخش (در نمونه‌ی فعلی، لوازم جانبی موبایل) است که از طریق ویزیتور (نماینده‌ی فروش سیار) سفارش می‌گیرد. هر ویزیتور با اکانت مخصوص خودش وارد می‌شود، لیست قیمت را می‌بیند، برای مشتری‌های خودش سفارش ثبت می‌کند، بازدیدهایش را همراه با موقعیت GPS ثبت می‌کند و یادآور می‌گذارد. ادمین از یک پنل مدیریت جدا، به داده‌ی همه‌ی ویزیتورها دسترسی دارد، کالاها را مدیریت می‌کند و آمار کلی می‌گیرد. تمام داده‌ها بین دستگاه‌ها از طریق Firebase Realtime Database هماهنگ می‌شوند و اپ به‌طور کامل آفلاین هم کار می‌کند.

## امکانات

### لیست قیمت و کالاها
- افزودن دستی کالا یا Import گروهی از فایل اکسل
- خواندن قیمت از روی عکس/اسکرین‌شات با OCR (سرویس OCR.space)
- هر کالا دو قیمت دارد: نقدی و چکی، به‌همراه حداکثر مدت چک (به ماه)
- گالری عکس برای هر کالا؛ تصاویر پیش از آپلود در مرورگر فشرده و روی ImgBB هاست می‌شوند
- خروجی اکسل و چاپ لیست قیمت

### سفارش و فاکتور
- سبد خرید، انتخاب مشتری از لیست، انتخاب روش پرداخت (نقدی/چکی)
- صدور فاکتور با شماره پیگیری؛ خروجی فاکتور به‌صورت تصویر (html2canvas)
- ارسال فاکتور از طریق واتساپ، هم برای مشتری و هم برای مدیر
- ویرایش/لغو/حذف سفارش، فیلتر بر اساس تاریخ شمسی، خروجی اکسل سفارش‌ها

### مشتریان
- هر ویزیتور فقط مشتریان خودش را می‌بیند و مدیریت می‌کند (نام فروشگاه، مالک، تلفن، یادداشت)
- تماس و واتساپ مستقیم از روی کارت مشتری، مشاهده‌ی تاریخچه‌ی سفارش‌های هر مشتری، خروجی اکسل مشتریان

### بازدید و نقشه
- ثبت بازدید با وضعیت (بازدید شد / سفارش گرفته شد / بدون سفارش / مشتری جدید / نیاز به پیگیری) به‌همراه موقعیت GPS
- امکان ثبت خودکار مشتری جدید از دل یک بازدید
- نمایش بازدیدها روی نقشه (Leaflet)، هم در حساب ویزیتور و هم در پنل ادمین (با فیلتر روی همه‌ی ویزیتورها)

### یادآور
- یادآور برای هر مشتری با تاریخ سررسید شمسی
- بنر «کارهای امروز» در بالای صفحه و نشان (badge) اعلان روی زنگ

### پنل مدیریت
- ورود جداگانه با رمز ادمین
- تب‌های کالاها، ویزیتورها، مشتریان، سفارش‌ها، بازدیدها و آمار
- افزودن/حذف ویزیتور و تغییر رمز هرکدام
- آمار به تفکیک هر ویزیتور (تعداد بازدید، سفارش، مشتری و مبلغ فروش) با فیلتر بازه‌ی تاریخ

### همگام‌سازی و آفلاین
- ذخیره‌ی محلی در `localStorage` + همگام‌سازی خودکار با Firebase Realtime Database (با تأخیر ۱.۵ ثانیه‌ای پس از هر تغییر)
- Merge دوطرفه بر اساس جدیدترین `updatedAt` برای هر رکورد، همراه با منطقی برای تشخیص تغییر هم‌زمان از یک دستگاه دیگر تا داده‌ای گم نشود
- صف آفلاین: تغییرات هنگام قطعی اینترنت ذخیره و پس از اتصال مجدد ارسال می‌شوند
- Service Worker: خودِ `index.html` همیشه از شبکه گرفته می‌شود (network-first) تا آپدیت فوراً اعمال شود؛ فقط آیکون‌ها و مانیفست کش می‌شوند

### PWA و نصب
- قابل نصب روی صفحه‌ی اصلی موبایل/دسکتاپ (آیکون در دو اندازه‌ی ۱۹۲ و ۵۱۲، با نسخه‌ی maskable)
- تم تیره/روشن با ذخیره‌ی ترجیح کاربر
- تقویم شمسی (جلالی) در تمام تاریخ‌های اپ

### پشتیبان‌گیری
- ادمین: بکاپ کامل و بازگردانی کامل (دانلود/آپلود فایل JSON، با امکان انتخاب بخش‌های موردنظر)
- ویزیتور: «بکاپ من»، فقط برای داده‌های خودش

## تکنولوژی‌ها

| بخش | ابزار |
|---|---|
| رابط کاربری | HTML/CSS/JavaScript خالص (بدون فریم‌ورک)، فونت Vazirmatn |
| نقشه | Leaflet.js |
| اکسل | SheetJS (xlsx) |
| خروجی فاکتور به تصویر | html2canvas |
| دیتابیس و همگام‌سازی | Firebase Realtime Database |
| OCR قیمت | OCR.space API |
| هاست عکس کالاها | ImgBB API |
| آفلاین و نصب | Service Worker + Web App Manifest |

هیچ build step یا `package.json`ای در پروژه نیست؛ همه‌ی کتابخانه‌ها مستقیم از CDN لود می‌شوند.

## ساختار فایل‌ها

```
pwa2-main/
├── index.html                  # کل اپ: HTML + CSS + JS (تنها فایل منطق برنامه)
├── manifest.json                # مانیفست PWA (نام، آیکون‌ها، رنگ تم)
├── sw.js                        # Service Worker (کش و حالت آفلاین)
├── icon-192.png
├── icon-192-maskable.png
├── icon-512.png
├── icon-512-maskable.png
└── apple-touch-icon.png
```

## راه‌اندازی

نصب یا بیلدی لازم نیست؛ کافیست فایل‌ها را روی یک وب‌سرور استاتیک سرو کنید:

```bash
# یکی از این‌ها را برای اجرای محلی انتخاب کنید
npx serve .
python3 -m http.server 8080
```

برای انتشار، هر هاست استاتیک مناسب است (GitHub Pages، Netlify، Vercel، Cloudflare Pages و…).

> **نکته:** چون اپ از Service Worker استفاده می‌کند، برای فعال‌شدن کامل حالت آفلاین و قابلیت نصب باید روی HTTPS (یا `localhost`) سرو شود؛ باز کردن مستقیم فایل با `file://` این دو قابلیت را غیرفعال می‌کند.

## تنظیمات

همه‌ی تنظیمات پروژه ابتدای بخش `<script>` در `index.html` (شیء `CFG`، نزدیک خط ۱۵۹۴) قرار دارند:

| کلید | کاربرد |
|---|---|
| `adminPw` | رمز ورود به پنل مدیریت |
| `fbApiKey` / `fbDbUrl` | کلید و آدرس پروژه‌ی Firebase Realtime Database |
| `ocrKey` | کلید سرویس OCR.space |
| `imgKey` | کلید ImgBB برای آپلود عکس کالاها |
| `adminWA` | شماره واتساپی که فاکتورها به آن ارسال می‌شود |
| `shopName` | نامی که در هدر لیست قیمت و متن فاکتور نمایش داده می‌شود |

**نکته‌ی امنیتی:** چون اپ کاملاً کلاینت‌ساید است، همه‌ی این مقادیر — از جمله رمز ادمین — داخل کد و در دسترس هرکسی است که Source صفحه را ببیند. عمومی‌بودن کلید Firebase طبیعی است (امنیت واقعی باید از طریق Firebase Rules تأمین شود)، اما رمز ادمین صرفاً یک قفل ساده‌ی سمت کلاینت است، نه یک احراز هویت واقعی. اگر قرار است این ریپازیتوری جایی عمومی/پابلیک شود، بهتر است پیش از آن این مقادیر را تغییر دهید یا از طریق Firebase Rules محدودشان کنید.

## نقش‌ها و ورود

- **ویزیتور** — از صفحه‌ی انتخاب ویزیتور، نام و رمز خودش را وارد می‌کند. فقط به لیست قیمت (برای سفارش‌گیری) و به مشتریان، سفارش‌ها، بازدیدها و یادآورهای خودش دسترسی دارد.
- **ادمین** — از دکمه‌ی «مدیریت» با رمز جداگانه وارد می‌شود. کالاها را مدیریت می‌کند، ویزیتور اضافه/حذف می‌کند و به داده و آمار همه‌ی ویزیتورها دسترسی دارد.

ویزیتور جدید فقط از پنل مدیریت، تب «ویزیتورها»، اضافه می‌شود.

## محدودیت‌های شناخته‌شده

- بک‌اند اختصاصی وجود ندارد؛ Firebase Realtime Database تنها منبع داده‌ی ابری است.
- تست خودکار (unit/e2e) در پروژه نیست.
- رابط کاربری فقط فارسی و راست‌به‌چپ است.
- رمز ادمین و کلیدهای API به‌صورت متن ساده در کد قرار دارند (به بخش «تنظیمات» بالا نگاه کنید).
